Visualizer Thesis

The first job is spatial intuition.

People should understand the whole network in one glance: headwaters, confluence, reservoirs, delivery zones, and cumulative losses. The second job is consequence: every node should expose what it contributes and what it constrains.

Primary Questions

  • Where is water being lost between origin and demand?
  • Which dam policy improves power without starving irrigation?
  • Which communities miss drinking water minimums first?

Design Direction

  • River atlas aesthetic, not dashboard sludge
  • Node-first network view with scenario overlays
  • Time slider to show daily and monthly movement

Core Views

One engine, four ways to read it.

1. Basin Network

The canonical graph view with nodes, edges, losses, and hover details. This is where operators form intuition fast.

2. Sector Balance

Stacked daily and monthly charts for irrigation, drinking water, and spill so tradeoffs are obvious rather than buried in tables.

3. Dam Operations

Storage trajectories, release plans, minimum storage alerts, and power generation yield for each controlled reservoir.

4. Scenario Compare

Side-by-side views of baseline vs optimized policies, with delta labels for unmet demand, food output, and hydropower.

Data Contract

Outputs the visualizer should expect from Rust.

Per time step / per node

  • inflow, outflow, local inflow
  • reservoir storage and release
  • drinking water delivered vs minimum
  • irrigation delivered and food produced
  • hydropower generated vs target

Per edge

  • upstream sent flow
  • loss fraction configured
  • absolute water lost
  • delivered downstream flow

For optimization mode

  • objective score and weights
  • constraint violations by severity
  • policy deltas from baseline
  • top drivers of improvement or harm

Build Plan

Phased path from hackathon demo to serious tool.

01

Hackathon demo

Render the static network, animate daily flow, and expose hover cards for nodes and edges using the MVP scenario JSON.

02

Scenario playback

Add a time scrubber, sector charts, and a baseline results loader from `nrsm-cli` output.

03

Optimizer compare

Show side-by-side runs, objective deltas, and policy comparison for release schedules and delivery shortfalls.

04

Decision support

Flag tension points automatically: loss hot spots, reservoir stress, unmet demand clusters, and sector tradeoff warnings.
